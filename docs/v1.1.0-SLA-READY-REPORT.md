## SLA-1 · SSO Google OAuth (✅)

**Archivos creados/modificados:**
- `backend/src/services/jwt.service.ts` — sign/verify, TTL 8h, secret ≥32 chars
- `backend/src/services/email.service.ts` — Resend SDK, 3 templates
- `backend/src/routes/auth-google.routes.ts` — flujo OAuth completo
- `backend/src/server.ts` — registrado plugin + tenant

**Vars requeridas (`.env` prod):**
```
GOOGLE_OAUTH_CLIENT_ID=xxx.apps.googleusercontent.com
GOOGLE_OAUTH_CLIENT_SECRET=GOCSPX-xxx
GOOGLE_OAUTH_CALLBACK_URL=https://ams.tudominio.cl/api/auth/google/callback
GOOGLE_OAUTH_ALLOWED_DOMAINS=tucliente.cl,otrocliente.cl  # opcional
JWT_SECRET=<random 32+ chars>
JWT_TTL_HOURS=8
RESEND_API_KEY=re_xxx        # opcional, no-op si vacío
RESEND_FROM=alerts@tudominio.cl
```

**Backward compat:** Si `GOOGLE_OAUTH_CLIENT_ID` está vacío, los routes hacen
no-op y el login email/pwd sigue funcionando.

## Pendientes 🔴 OPERACIONALES (NO de código)

Estos son los únicos bloqueantes para deploy real a un cliente:

1. **Comprar dominio productivo** — sugerido: `.cl` por Nic Chile (~CLP 9K/año)
2. **DNS A record:** `ams.tudominio.cl` → IP del VPS Hetzner
3. **Activar Budget HARD CAP** Google Cloud Billing (no soft alert)
4. **Limpiar 8 de 9 tarjetas** en Google Billing (dejar solo 1)
5. **Deploy real al VPS Hetzner** con `bash deploy.sh prod`
6. **Setup rclone backup remoto** (B2 o S3) — script ya en `stack/scripts/`
7. **Activar cron de backup** en VPS — script ya provee
8. **Crear email soporte@tudominio.cl** + redirección a tu inbox
9. **(Opcional) Crear Google OAuth Credentials** en Cloud Console para SSO
10. **(Opcional) Slack webhook** para alertas
